The ultimate starter kit for building, testing, and publishing professional **Laravel** packages. Designed with best practices in mind, this boilerplate includes everything you need to go from idea to Packagist.

---

## 🚀 Why This Boilerplate?

Building a Laravel package from scratch can be overwhelming. You have to set up service providers, configuration files, migrations, and test environments. **Laravel Package Boilerplate** automates this process, so you can focus on writing code.

### Key Features:
- **Service Provider Pre-configured:** Automatic discovery and registration.
- **Config & Migrations:** Ready-to-use patterns for publishing assets.
- **Testing Suite:** Integrated with **Pest/PHPUnit** and **Orchestra Testbench**.
- **GitHub Actions Ready:** CI/CD workflows for linting, testing, and automated releases.
- **PSR-4 Autoloading:** Modern, standard-compliant directory structure.
- **SEO Optimized Docs:** Clean, readable README with professional badges.

---

## 🛠️ Installation

```bash
# Clone the repository
git clone https://github.com/AhmarHussain/laravel-package-boilerplate.git my-package

# Navigate to the directory
cd my-package

# Install dependencies
composer install
```

---

## 📖 Getting Started

1. **Search & Replace:** Replace `Stackaura\Skeleton` with your namespace and `laravel-package-boilerplate` with your package name.
2. **Implement Logic:** Add your code to the `src/` directory.
3. **Register Assets:** Use the `SkeletonServiceProvider` to load migrations, routes, or views.
4. **Write Tests:** Add your test cases in the `tests/` directory.
